# SQL Business Reporting & Exploratory Data Analysis Project

## Project Overview
This project demonstrates my SQL skills by working with curated data from the **Gold Layer views** of a data warehouse to create **business-ready customer and product reports** and perform **exploratory data analysis (EDA)** to answer key business questions.

The main goal of this project was to transform trusted, analytics-ready data into meaningful insights that support business decision-making.

---

## Objectives
- Use data from **Gold Layer views** as the primary source for analysis
- Build **business-focused reports** for customers and products
- Perform **exploratory data analysis (EDA)** to identify trends, patterns, and opportunities
- Answer common business questions using SQL
- Showcase practical SQL skills used in real-world analytics and reporting

---

## Data Source
The project uses data from the **Gold Layer** of the data warehouse whch includes customers, Products and fact table views, and contains cleaned, transformed, and business-ready views.

These views serve as the final analytical layer and are ideal for:
- Reporting
- Dashboarding
- Business performance analysis
- Ad hoc analytical queries

---

## Project Scope
This project includes two main components:

### 1. Business Reporting
Created SQL queries to generate business-required reports such as:

- **Customer Report**
  - Customer details and segmentation
  - Purchase behavior analysis
  - Revenue contribution by customer
  - Order frequency / customer activity insights

- **Product Report**
  - Product-level sales performance
  - Top-performing and low-performing products
  - Category-wise analysis
  - Revenue and quantity trends by product

### 2. Exploratory Data Analysis (EDA)
Performed SQL-based EDA on the Gold Layer views to answer business questions such as:

- Which customers generate the highest revenue?
- Which products or categories perform best?
- What are the sales trends over time?
- Which products have low sales performance?
- Are there any patterns in customer purchasing behavior?
- Which segments contribute the most to overall business performance?

## Key SQL Concepts Used
This project demonstrates hands-on use of:

- `SELECT`, `WHERE`, `GROUP BY`, `ORDER BY`
- Aggregate functions: `SUM()`, `AVG()`, `COUNT()`, `MIN()`, `MAX()`
- Joins: `INNER JOIN`, `LEFT JOIN`
- Common Table Expressions (**CTEs**)
- Window functions:
  - `ROW_NUMBER()`
  - `RANK()`
  - `DENSE_RANK()`
  - Running totals / partitioning
- Conditional logic with `CASE WHEN`
- Date-based analysis
- Subqueries
- Data filtering and business-rule driven transformations
